## 1. 台本内CTA設計の原則（本筋を崩さずに差し込む技法）

### 1-1. CTAの3層構造
台本内のCTAは以下の3層で構成されている：

```
① 否定からの反転：「必勝法はない」→「ただ、裏設定3選はコミュニティで配布してる」
② サバのツッコミで宣伝臭を中和：「え、あるんかい！！」
③ 本筋に戻す一文で閉じる：「まずは今日の基本をしっかり押さえてからね」
```

### 1-2. CTA挿入の判断基準
- **本筋の主張と矛盾しないこと**：「必勝法はない」と言いつつ「裏設定はある」→ 本筋（視聴者満足が本質）を否定しない範囲
- **掛け合いの笑いで中和すること**：宣伝単体で挿入しない。必ずサバの反応で笑いに変換する
- **1〜3行で完結すること**：長いCTAは離脱を招く。差し込んですぐ戻る

### 1-3. CTA挿入のパターン一覧（台本で実証済み）

| パターン | 挿入タイミング | 訴求内容 | 中和方法 |
|---|---|---|---|
| 否定→反転 | 「必勝法はない」の直後 | 裏設定3選をコミュニティで配布 | サバ「あるんかい！！」 |
| 解説→補足 | リサーチツールの話題 | ツール1ヶ月限定トライアル | サバ「めちゃくちゃ欲しい！！」 |
| ストック解説→実物 | データをまとめる重要性の後 | リサーチスプシをコミュニティで配布 | 画面録画挿入＋サバ「便利なものが…！」 |

### 1-4. CTA挿入時の希少性・緊急性の演出
- 「今だけ特別に」＋「1ヶ月限定」で希少性と緊急性を同時に付与
- 「基本が身についた人向け」を「今だけ特別に」に変更 → 条件付きから期間限定へ転換し、即行動を促す

## 2. 視聴者意見の先回り代弁（サバの役割設計）

### 2-1. サバ＝視聴者の心理的抵抗を言語化する装置
サバの最重要機能は「視聴者が画面の向こうで感じている疑問・不満・不安」を先に言葉にすること。

### 2-2. 先回り代弁のパターン

| パターン | サバのセリフ例 | 効果 |
|---|---|---|
| 「別にいいじゃん」型 | 「それの何がまずいんです？」 | 視聴者の抵抗を先に拾って、イルカが論理で潰す |
| 「早く教えて」型 | 「結局どうやってジャンルを選べばいいの？」 | 視聴者の「早く実践見せて」を代弁 |
| 「自分もそう」型 | 「あー、それめちゃくちゃある…調べたのに活かせてないやつ」 | 視聴者のあるあるを言語化して共感を生む |
| 「期待と裏切り」型 | 「さっきのツール使うんじゃないんですか！？」 | 視聴者の予想を代弁→イルカの回答で「なるほど」を作る |

### 2-3. 先回り代弁の価値判断
- 視聴者が疑問を抱く**前に**解消することで、離脱を防止
- サバが代わりにツッコむことで、視聴者は「わかってくれてる」と感じる
- これは単なるギャグではなく、**離脱防止とテンポ維持を両立する構造的な設計**

## 3. 掛け合い台本の設計思想

### 3-1. イルカが詰める→サバが自分で答えを出す構造

```
イルカ：「リサーチをサボった結果、どうなると思います？」
サバ：「えっと…伸びない…？」
イルカ：「それから？」
サバ：「改善点もわからない…」
イルカ：「それから？」
サバ：「メンタル崩壊して…YouTubeつらくなる…」
イルカ：「はい、全部自分で言えましたね。」
```

- イルカが答えを言うのではなく、**サバに自分で言わせる**
- 視聴者も一緒に考えさせられるので、教育効果が高い
- サバの「自分で言って自分で傷ついた」で笑いに変換

### 3-2. 「肩透かし→納得」のギャグ構造

```
サバ：「ツールは…？」
イルカ：「YouTubeです！」
サバ：「え？」
イルカ：「え？？」
サバ：「YouTubeですよね？」
イルカ：「YouTubeです。」
```

- 視聴者の期待（高度なツール）を裏切り、シンプルな答えで返す
- 繰り返しのリズムで笑いを生む
- その後に「なぜYouTubeから始めるべきか」の論理を展開して納得させる

### 3-3. ギャグ→即実践の切り替えテンポ

```
サバ：「で、ジャンルをメモったはいいけど、次は何すればいいんですか？」
イルカ：「はい、先ほど洗い出したジャンルをYouTubeの検索窓にぶち込んでください。」
サバ：「え、もうやるの！？」
イルカ：「はい、もうやります。」
```

- 理論パートから実践パートへの切り替えを、ギャグのテンポで処理する
- 冗長な繋ぎ（10行）を圧縮（4行）し、「視聴者が実践を見たいタイミング」と同期させる

## 4. 仮想敵の設計（ポジショニングとファン化の両立）

### 4-1. 仮想敵は「人物」ではなく「構造」に寄せる
- **NG**：「○○さんの動画はダメ」（人物攻撃）
- **OK**：「リサーチスキルを持っていないから手順を出せない構造」（構造批判）

### 4-2. 仮想敵の描写は行動まで具体的に書く
- **NG**：「このジャンルやってください」としか言えない
- **OK**：「他の発信者の情報を自分の発信に横展開して、このジャンルやってください」としか言えない
- 仮想敵が**何を・どうやって・なぜそうなるのか**の構造まで書くことで、批判に説得力が生まれる

### 4-3. 仮想敵→自分のポジション→実績の3ステップ

```
① 仮想敵の構造を暴く：「リサーチスキルがないから手順を出せない」
② 自分のポジションを示す：「僕はこの判断基準を持っている」
③ 実績で証明する：「だからジャンル選定で外さなくなった」
```

### 4-4. 親友理論との組み合わせ
- 仮想敵を批判する前に、「僕も初心者でYouTube始めたての頃、同じことを感じてました」と共感を入れる
- 「上から批判する専門家」ではなく「同じ痛みを知った上で乗り越えた同志」のポジション

## 5. 期待値の設計（フレームワーク紹介前のタメの作り方）

### 5-1. 期待値を上げる4ステップ

```
① サバの困惑：「結局何を基準に選べばいいんですか？感覚？勘？」
② イルカのタメ：「誰でも再現できる形で判断できる方法がある」
③ 汎用性の提示：「YouTube、Instagram、TikTok、X…どのSNSでも使える」
④ 実績の自己開示：「これを学んでから売上ゼロ→月50万円まで跳ね上がった」
```

### 5-2. 再現性の強調が最も視聴者のマインドを上げる
- 「才能もセンスも関係ない」
- 「やり方さえ知っていれば、誰でも同じ精度で判断できる」
- 視聴者が最も欲しいのは「自分にもできる」という確信

### 5-3. 実績の自己開示はフレームワークの「証拠」として機能する
- 「僕自身、このフレームワークを学ぶ前は売上ゼロの月が続いていた」→ 過去（親友理論）
- 「使い始めてから一気に月50万円」→ 現在（親友理論）
- サバの「ゼロから50万！？」で視聴者の驚きを代弁

## 6. 教育かけの技法（マインドセット教育の段階設計）

### 6-1. リサーチ投資教育の3段階

| 段階 | 内容 | 教育効果 |
|---|---|---|
| 1. リサーチ＝最強 | 「はっきり言います。リサーチが一番の裏技です」 | リサーチの地位を引き上げる |
| 2. 無料でできる | 「やればやるほど、無料でデータが貯まっていく」 | 「やらない理由がない」の論理 |
| 3. 投資マインド | 「お金を払ってでもリサーチの精度を上げたい」→「無料で全力出せない人が有料ツールを使いこなせるわけがない」 | 無料→有料への投資マインドの種まき |

### 6-2. 教育かけの本質
- 直接「お金を使え」とは言わない
- 「無料でできることに全力を出す」→「その延長線上に有料がある」という順序
- 将来の商品購入への心理的ハードルを、教育パートの中で自然に下げる

## 7. 最悪の未来を描く技法（行動促進のための恐怖設計）

### 7-1. 段階的な転落プロセスで描写する
- **NG**：「リサーチしないとダメになります」（抽象的）
- **OK**：「気づかずリサーチした気になって → そのまま需要とズレた動画を量産して → 伸びない原因すらわからないまま消耗する。最悪のルートです」

### 7-2. 転落の各段階に具体的な動詞を使う
- 「気づかず」→ 認知の欠如
- 「リサーチした気になって」→ 誤った安心
- 「需要とズレた動画を量産」→ 行動の方向ミス
- 「伸びない原因すらわからない」→ 改善不能
- 「消耗する」→ 最終的な結末

## 8. 比喩の具体性ルール（一発で絵が浮かぶ比喩）

### 8-1. 抽象的な状況を「一発で絵が浮かぶ比喩」に変換する

| 抽象表現 | 具体的な比喩 |
|---|---|
| 激戦区に突っ込む | 料理初心者なのに、いきなりイタリアでレストラン出す |
| 需要ゼロの場所で発信する | 北極でアイスを売る |
| 1ジャンルだけ見て即決する | 転職で1社だけ見て即決する |

### 8-2. 比喩の後にサバのリアクションで定着させる
- 比喩 → サバの「本場のシェフたちにボコボコにされて終わり」→ 視聴者の頭に映像が残る
- 比喩単体で終わらせず、掛け合いで膨らませることで記憶に定着する

## 9. 具体性の執筆ルール（NGパターンと修正パターン）

### 9-1. 動詞の具体化（抽象的な変化動詞を避ける）

| NG | OK | 理由 |
|---|---|---|
| おすすめが趣味に**なる** | おすすめが趣味に**寄る** | 「なる」は曖昧。「寄る」は実際の現象を描写 |

### 9-2. 指示語の排除（「何を・どこに・どうする」を省略しない）

| NG | OK | 理由 |
|---|---|---|
| YouTubeの検索窓に**ぶち込んで** | **先ほど洗い出したジャンルを**YouTubeの検索窓にぶち込んで | 「何を」が抜けている |

### 9-3. 時期・状況の具体化（曖昧な時期表現を避ける）

| NG | OK | 理由 |
|---|---|---|
| 僕も**初心者の頃** | 僕も**初心者でYouTube始めたての頃** | 何の初心者か、いつの話か不明 |

### 9-4. 仮想敵の行動描写の具体化（中身まで書く）

| NG | OK | 理由 |
|---|---|---|
| 「このジャンルやってください」としか言えない | 「他の発信者の情報を自分の発信に**横展開して**、このジャンルやってください」としか言えない | 仮想敵が実際にやっていることの構造が見えない |

### 9-5. 全体原則
- 読者・視聴者が「頭の中で映像を再生できるか」が判断基準
- 抽象的に濁す表現は、AIっぽさ・テンプレ感を生む
- 具体的な動詞・対象・時期を書くことで、「人間が書いた文章」の質感が出る

## 10. NGワード管理（AIっぽい表現の排除）

### 10-1. 「地雷」の排除
「地雷」はAIが多用する表現のため、全面的に使用禁止。

| 文脈 | 代替表現 |
|---|---|
| 危険な選択 | ハズレ、やばいやつ、危険、罠、落とし穴 |
| 危険な場所 | 危険地帯、事故物件 |
| 危険を察知する | ハズレを見抜く |
| 踏んではいけないもの | 爆弾 |

### 10-2. AIっぽい表現を避ける判断基準
- 「よく見る言い回し」＝AIが学習データから高頻度で出力する表現
- 人間が会話で自然に使う言葉かどうかで判定する
- 迷ったら「口に出して違和感がないか」でチェック

## 11. 編集指示の組み込み方

### 11-1. フォーマット
```
【編集指示：ここで○○を挿入。△△ありで、□□する形で収録。○○が伝わるように】
```

### 11-2. 編集指示に含めるべき要素
- **何を映すか**：リサーチスプシの画面録画
- **加工指示**：モザイクあり
- **動きの指示**：少しスクロールして動かす
- **伝えたい印象**：実際に使われている感、情報量の多さ

## 12. 冗長な繋ぎの排除ルール

### 12-1. 判断基準
- 視聴者が「早く次に進んで」と感じるタイミングで繋ぎが長い＝冗長
- 特にパート切り替え（理論→実践）は最短で切り替える

### 12-2. 圧縮の実例

**Before（10行）：**
```
うわ…なんか良いこと言って締めた…
当然です。私はイケメン教育系イルカですからね。
勝手に肩書き作るなｗｗ
で、リサーチが大事なのはわかったけど…
（さらに5行の繋ぎ）
```

**After（4行）：**
```
うわ…なんか良いこと言って締めた…
で、リサーチが大事なのはもう十分わかりました！具体的に何からやればいいんですか？
はい、ここからは実践です！
ではジャンルリサーチ、実践スタート！
```

### 12-3. 原則
- ギャグの余韻は1〜2行で切る
- 視聴者の「次は？」と台本の「次は？」を同期させる
- サバが視聴者の「早く教えて」を代弁することで自然に切り替える
