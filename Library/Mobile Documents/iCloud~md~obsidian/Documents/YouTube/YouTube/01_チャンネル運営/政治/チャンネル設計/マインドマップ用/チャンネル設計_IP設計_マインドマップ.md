        - 掛け合い/役割分担：IPの魅力を引き出す相方やフォーマット

## 0-1. なぜ今の時代にIP設計が不可欠なのか（信頼できないコンテンツ問題）

- なぜIP設計が不可欠なのか
    - YT規制強化の背景
        - 2025年7月 量産型規制強化
        - 信頼できないコンテンツで収益剥奪
        - 量産型は淘汰対象
        - 2026年1月 CEO品質重視を発表
    - IPがないchの末路
        - 誰が運営しても同じに見える
        - 量産型と判定されるリスク
        - 見る理由がない
        - コミュニティが作れない
    - IPがあるchの強み
        - この人を見たい理由が生まれる
        - なぜやっているかが伝わる
        - 成功も失敗も追いたくなる
        - 次の投稿が気になる
    - YTが本当に求めていること
        - 各chを独自のコミュニティに
        - 運営者の視点と意見
        - 誰が何を考えて発信しているか
        - AIに代替できない人間の思考

- 解説ポイント：なぜ「今」IP設計を真剣にやる必要があるのか
    - 結論：IP設計ができていないchはYouTubeから淘汰される時代に入った
    - ① YouTubeの規約変更：量産型コンテンツの一斉排除
        - 2025年7月15日 ガイドライン改定：量産的で反復的なコンテンツの収益化制限を強化
        - 参照：02_SNS運用/X/みこさわX/商材/台本/信頼できないコンテンツ.md
        - 参照：https://support.google.com/youtube/answer/1311392?hl=ja
        - YouTube側の意図
            - AI生成動画の急増→低品質コンテンツ大量投稿→視聴者が信頼できない状況に
            - サーバーコストの増大：低品質動画の蔓延でコスト増
            - 広告主離れの防止：低品質動画に広告→ブランド価値低下→プラットフォーム信頼性低下
            - 2026年1月 CEOニール・モーハンが公式で「品質重視」を発表
        - YTの方向性：量産型を排除し、本物のクリエイターを残す
    - ② 「信頼できないコンテンツ」の本当の意味
        - 英語原語：Inauthentic content＝「本物ではないコンテンツ」
        - 嘘や不正確ではなく「量産型で独自性がなく誰が作っても同じに見える」という意味
        - YTが見ているのは動画の中身ではなくch全体の見え方
            - 機械音声を使用している
            - 投稿頻度が異常に多い
            - サムネとタイトルの型が毎回同じ
            - 動画構成が毎回ほぼ同じ
            - 誰が運営しているかわからない
            - 運営者独自の一次情報や意見がない
            - 誰でも作れそうな内容
        - これらが重なると真面目に作っていても「量産型」判定→収益剥奪

    - ③ なぜIP設計がこの問題の解決策になるのか
        - YTの方向性：「各chを独自のコミュニティにしたい」
        - コミュニティ＝運営者の考え方や価値観に共感した人が集まる場
        - IP＝単なるキャラデザインではない。「この人（ch）を見る理由」を作ること
        - IPの6要素（政治chの具体例付き）
            - 誰がやっているか：脱力塾長を通して「減税を本気で願っている人」が伝わる
            - なぜやっているか：「煽りではなく整理で、視聴者の不安を消したい」という信念
            - この人が歩む先の未来を見たい：減税ミッション/減税ゲージで連載化
            - 次の投稿が気になる：予測→答え合わせで「塾長の予測は当たるのか？」
            - 成功も失敗も含めて追いたい：予測が外れたらギャグ化＋訂正＝信頼が積み上がる
            - 独自の視点と意見がある：事実ではなく「解釈」を聞きに来る
        - IPがないchで起きること
            - 誰が運営しても同じに見える→量産型判定→収益剥奪リスク
            - 見る理由がない→他chで代替可能→登録理由なし
            - コミュニティが作れない→コメント欄が活性化しない
            - 次見る理由がない→毎回完結→継続視聴されない
        - IPがあるchで変わること
            - 「この人の解説が見たい」→唯一無二の存在
            - 「運営者の考えが見える」→独自性判定→収益安全
            - 「共感した人が集まる」→コメント欄がコミュニティ化
            - 「次が気になる」→連載的に追いかけたくなる→継続視聴UP
    - ④ 具体例：批判系chがIP設計で負けている理由
        - 批判系ch（IPが弱い/ない）
            - https://www.youtube.com/@kokumin-minsyu%E6%AD%A3%E8%AB%96%E3%83%91%E3%83%B3%E3%83%81/videos
            - https://www.youtube.com/@yukkuriseiji
            - https://www.youtube.com/@yukkurinewsch
            - https://www.youtube.com/@%E5%9B%BD%E6%B0%91%E3%83%9E%E3%82%B8%E3%82%AE%E3%83%AC%E5%85%9A
        - 共通するIP設計上の問題
            - 「誰がやっているか」がわからない→見る理由がない
            - 「なぜやっているか」が見えない→批判が目的に見える
            - 「次の投稿が気になる」仕掛けがない→毎回完結で見なくてもいい
            - 「独自の視点」が弱い→ニュースの感想/反応止まり
            - テンプレ的な構成→量産型判定リスク
        - ベンチマークchはIPが比較的ある
            - https://www.youtube.com/@Seiji_Muteki_Oji →ギャグ×政治の独自切り口（ただしIPの深さはまだ弱い）
        - 今回の政治chではベンチマークchの「IP設計の弱さ」を潰す→差別化の核心
    - ⑤ ディレクターに理解してほしい核心

## 0-3. チャンネル設計で洗い出す9項目：各項目の詳細

- 9項目の詳細（STEP1〜5で何を洗い出すか/なぜ必要か/政治chではどうしたか）
    - 【1】視聴者理解と時代の変化（すべての起点）
        - 話すこと：視聴者を取り巻く環境がどう変わったかを最初に整理する。全設計の土台
        - 視聴者が求めるものは時代で変わる。3年前に伸びたやり方が今は伸びない
        - 洗い出す内容
            - 時代の変化：環境が「以前と比べて」どう変わったか
                - 例：3年前は「叩き/炎上/暴露」で伸びた→今は「批判系に疲れている」
                - 例：「善悪ジャッジ疲れ」が広がっている
                - 例：インフレ・増税→「怒り」より「不安の解消（安心感/判断軸）」を求めている
            - 視聴者の心理状態：年代・性別ではなく「今どんな気持ちでYTを開いているか」で定義
                - 例：不安を抱えている/メディアを信用していない/でも自分で調べるのは面倒
                - 例：「学んでいる自分」でいたい。娯楽を見ている自分ではなく
            - 面白いの定義：視聴者にとっての「面白い」は「笑い/娯楽」とは限らない
                - 例：政治系の「面白い」→安心感/腹落ち感/網羅感/構造が見える感/代弁してくれた感
        - なぜこれが最初なのか
            - 「面白いの定義」がズレたまま企画を作ると、視聴者が離脱する
            - 企画・サムネ・台本・IP設計すべての判断基準は、この視聴者理解から始まる
    - 【2】視聴者に足りないもの（現状のメディアで満たされていないニーズ）
        - 話すこと：「なぜこのchが必要なのか」の根拠。「足りないもの」を特定する
        - 「足りないもの」＝chの存在意義。そこを埋めるchは需要がある
        - 洗い出す内容
            - 「欲しいのに今のchでは得られない」もの
                - 例：構造を整理して全体像を見せてくれるch→断片的な切り抜きchばかり
                - 例：具体的な解決策や判断材料→不安を煽るだけで「どうすればいいの？」がない
                - 例：中立的な視点で俯瞰→TVは偏向報道、YTは叩き系ばかり
        - 具体例（政治ch）：批判系ch（@yukkuriseiji @yukkurinewsch等）は不安を煽るが「全体像」「解決策」を提供できていない→ここが「足りないもの」
    - 【3】成功の3つの要素と3つのハードル
        - 話すこと：「視聴者が満足する条件」と「離脱する壁」を先に設計する。企画の判断フィルター
        - 成功の3要素：視聴者が「見て良かった」と感じる3条件
            - なぜ3つに絞るか→判断基準は少ないほど使いやすい
            - 例（政治ch）：全体像の把握 / 未来の見通し / 判断材料の提供
        - 3つのハードル：視聴者が見る前/見続ける前に立ちはだかる壁
            - 設計段階で潰しておかないとそもそも見てもらえない
            - 例（政治ch）：政治＝難しそう / 解説ch＝ストレスになりそう / 勉強＝面倒くさい
        - 使い方：「3要素のどれを満たすか？」「ハードルのどれを解除するか？」で判断。両方に答えられない企画は出さない
    - 【4】価値提供と需要
        - 話すこと：「誰に、何を、なぜこのchで」を明確にする。曖昧だと「なんとなく良さそうな企画」しか出てこない
        - 誰に：属性ではなく心理状態で定義する
            - 例：「政治に不安があるがTVは信用できず、自分で調べるのは面倒で、整理してくれるchが欲しい層」
        - 何を：情報ではなく視聴後に残る感覚で定義する
            - 例：「安心感と判断軸を持ち帰れる体験」「結局こういうことかという腹落ち感が残る」
        - なぜこのchでなければいけないか：情報だけなら他でも手に入る。このchだけの体験を言語化
            - 例：「脱力した掛け合いで盗み聞きする感覚。気軽に見られるのに最後は論点が整理される」
            - →IPがあるからこそ成立する体験。IPがなければ「他のchでも同じ」
    - 【5】視聴者の希望する未来
        - 話すこと：「見続けた先に手に入れたい状態」を言語化する。サムネの「約束」はここから作る
        - 短期の希望（今日の1本で得たいもの）
            - 例：「今話題のあの件、結局どういうことなのか？」が安心してわかる
        - 長期の希望（見続けた先に手に入る状態）
            - 例：政治の全体像がわかる博識な自分 / 判断軸を持って振り回されない / 家族や友人に「詳しいね」と言われる
        - なぜ重要か：希望の未来がわかるとサムネの「約束」が明確になる。サムネ＝「見たらこの状態になれますよ」という約束
    - 【6】仮想敵と独自性
        - 話すこと：「何と戦っているのか」を明確にする。仮想敵が決まると立場＝独自性が自動的に決まる
        - 仮想敵のルール：人物/政党を敵にすると炎上リスク。構造・制度・環境を敵にすると安全
            - NG例：「〇〇党は悪い」「〇〇議員は嘘つき」→猛反発/通報
            - OK例：「解決策を出さない発信」「偏向報道の環境」「事実不明のまま断定する情報環境」
        - 独自性：仮想敵が決まると「うちのchはどう違うのか？」が自然に出てくる
            - 突飛なことではなく「ここが他と違う」と実感できる体験を設計する
        - 差別化の考え方：競合の強みは自chでもやる。競合の弱みを自chの強みで潰す
            - 例：BM（@Seiji_Muteki_Oji）のギャグ精度は高い（＝強み→うちもやる）。安心感/網羅感/解決策が弱い（＝弱み→ここを潰す）
    - 【7】企画とサムネイル
        - 話すこと：「〇〇感フレーム」でサムネと企画を設計する方法
        - 〇〇感フレーム：サムネは「出来事」ではなく「見た後に手に入る感覚」の約束
            - NG例：「〇〇党が〇〇を発表」→出来事の報告。見る理由がない
            - OK例：「〇〇 完全解説」→「網羅感」を約束
            - OK例：「なぜ〇〇なのか？」→「腹落ち感」を約束
        - 伸びるワードの分類（7カテゴリ）
            - 網羅系 / 問い系 / 安心系 / 結論系 / 実益系 / ツッコミ系 / 衝撃系
        - Good/NG企画の型
            - Good：「安心感＋実益」がある→見た後に判断材料や全体像が手に入る
            - NG：「不安だけ煽って解決策がない」→見た後に不安が増えただけで疲れる
        - 具体例で比較
            - BM（@Seiji_Muteki_Oji @user-i-love-TVstations @NKTofficial）→「完全解説」「全史」「なぜ〜」系が伸びている
            - 批判系（@yukkuriseiji @yukkurinewsch）→「炎上」「暴露」「闇」系は短期で伸びても長期で疲れさせる
    - 【8】競合の勝ち筋
        - 話すこと：競合は「模写」ではなく「構造」を抜く。表面を真似しても劣化コピーにしかならない
        - 「なぜ伸びているか」を構造で分解する
            - タイトル構成（感情ワード＋知的ワード＋対象）
            - サムネ構成（1メッセージ化/固有名詞/数字の使い方）
            - 冒頭30秒の約束の仕方
            - 動画全体の構成（結論→背景→利害→根拠→反対意見処理→要約）
            - 視聴後に残る報酬（感情＋理解＋語れるネタ化）
        - 競合の強みは取り込む：「伸びていること」はうちでもやる
            - 例：@Seiji_Muteki_Oji のテンポとギャグ×政治がスッキリわかる（報酬の二重化）
        - 競合の弱みは潰す：「満たされていないニーズ」をうちで満たす
            - 例：事実誤り→うちは一次情報/時系列を徹底
            - 例：判断材料が弱い→うちは「結局どうすればいいか」を必ず回収
            - 例：IPが弱い→うちはIPを作り込んで「この人を追いかけたい」を作る
    - 【9】IP（知的財産）の設計（最重要パート）
        - 話すこと：IP設計はchの生命線。「この人（ch）を見る理由」を作ること
        - IPとは何か（正しく理解すべき定義）
            - 「キャラデザイン」ではない。「視聴者がそのchを見続ける理由の構造」
        - IP6要素
            - ① 誰がやっているか（人格・スタンスが見える）
                - 視聴者は「どんな人がこれを言っているのか」を気にする
                - 例：脱力塾長→「普段はだらしないけど鋭い。減税を本気で願っている人」
                - →「この人の解説が見たい」になる
            - ② なぜやっているか（動機・使命が見える）
                - 「なぜ政治を解説する必要があると思っているのか」が伝わるか
                - 例：「煽りではなく整理で不安を消したい」「偏向報道に振り回されない判断軸を届けたい」
                - →「このchの目的に共感する」→コミュニティが生まれる
            - ③ この人が歩む先の未来を見たい（長期目標が見える）
                - 「どこに向かっているのか」を知っていて見届けたいと思えるか
                - 例：減税ミッション/減税ゲージ→「減税を実現するまでの過程を見届けたい」
                - →「応援したい」「最後まで見届けたい」になる
            - ④ 次の投稿が気になる（連載感がある）
                - 毎回完結だと「見なくていい理由」が発生する
                - 予測→答え合わせ：「前回予測したあの件、結局どうなった？」
                - 伏線→回収：「続報が来たら解説する」と予告
                - →毎回完結chは「情報取ったら離脱」。連載感chは「追いかける」
            - ⑤ 成功も失敗も含めて追いたい（人間味がある）
                - 完璧なキャラは追いかけにくい。失敗を見せると親近感が生まれる
                - 例：予測が外れたら→ギャグ化＋訂正→「外れを隠さない＝信頼」が積み上がる
                - →「完璧な解説者」より「鋭いけど外す、でも正直な人」の方がファンがつく
            - ⑥ 独自の視点と意見がある（AIに代替できない部分）
                - 事実の羅列ならニュース記事やAIで代替できる。YTで見る必要がない
                - 運営者の独自の解釈・分析・意見があるから「このchで見る価値」が生まれる
                - YTが公式で求めていることもこれ：「運営者独自の意見」「教育的価値」
                - 例：「塾長はこう解釈している」→視聴者は事実ではなく解釈を聞きに来る
        - なぜ掛け合い構造が必要か
            - 1人語りだと「授業されている感」が出る（ウーバーイーツ/一蘭と同じ心理）
            - →掛け合い形式で「他の人の会話を盗み聞きする感覚」にする
            - 役割分担：エンタメ担当（塾長）×信頼担当（シン）
            - 掛け合い構造自体がIPの一部。批判系にもBMにも本格的な掛け合いIPはない→差別化の核心
        - ギャップ設計：IPの魅力を最大化する方法
            - IPの魅力の本質は「ギャップ」
            - 例：脱力塾長→普段はコタツでゴロゴロ×解説は論点3点で鋭い整理→この落差がキャラの魅力
            - 例：「減税」への異常な執着→普段は脱力なのに減税の話で目の色が変わる→お約束化でファン定着

## 0-4. 最も重要な思考法：「長期の敗北条件」を先に潰す

- 長期で勝つための思考順序
    - STEP1 敗北条件を先に潰す
        - 信用毀損は回復不能
        - 根拠不足で信用が積めない
        - 規約違反でch削除
        - コメント欄荒れで離脱
        - 制作コスト高で継続不能
    - STEP2 信用の積み上げ設計
        - 毎回結論で回収する型
        - 出典と一次情報を残す
        - 間違いは隠さず訂正
        - 反対意見や例外も扱う
    - STEP3 短期の伸びを乗せる
        - ギャグとテンポのエンタメ感
        - サムネの強さでクリック率
        - 伸びるワードで訴求力
        - リッチデザインでTV特番感

- 解説ポイント：ch設計で最も差がつくのは「思考の順番」
    - 成功する人の思考順序
        - 1番目：「このchが長期で死ぬ条件」を先に潰す
        - 2番目：信用が積み上がる仕組みを固める
        - 3番目：安全な上にエンタメ/伸び施策を乗せる
    - 失敗する人の思考順序
        - 1番目：「短期で伸びる方法」を先にやる
        - 2番目：伸びなくなったら考える
        - 3番目：信用が壊れても「後で取り戻せる」と思う
    - 成功する人 vs 失敗する人の違い
        - 時間軸：長期の死ぬ条件→短期の伸び vs 短期で伸びる方法→長期は後回し
        - 主語の置き方：制度/構造を主語（安全） vs 人物叩き/炎上が主語（通報リスク）
        - 反証の有無：外れる条件/反対意見まで設計に入れる vs 断定するだけ→外れたら信用崩壊
        - 報酬設計：感情報酬＋理解報酬の二重化 vs 感情報酬（怒り/断罪）に偏る→疲れて習慣化しない
        - 資産化：設計をファイルに落として更新 vs その場の納得で終わり→ゼロスタート
    - 長期の敗北条件チェックリスト（先に潰す）
        - サムネ/タイトルで約束した価値を動画内で回収できていない（裏切られた感→信用を失う）
        - 主張に裏付けがなく「それっぽい話」に見える（根拠不足→信用が積めない）

        - 狙い：視聴者が「情報」ではなく"人（IP・運営者）"を追う理由
        - 1）唯一性
            - 脱力×論点整理の両立：政治を"重い授業"にせず、脱力しながらも論点だけを切り出して整理できる
            - 中立演出：政治批判を呆れた笑いに変換しつつ、最後は視聴者に学びを届ける（感情→理解の二段報酬）
            - 師弟の掛け合いで担保する信頼：塾長が脱線しても、シンが「根拠」「いつの話」「結論」を冷静に折檻して軌道修正（＝安心感）

            - ニュースの"再編集"：時系列・因果・利害で再編集して「結局こういうこと」を作る

            - 政党・人物ではなく"構造/制度/プロセス"を見る：人格攻撃を避け、システム構造視点で語る
            - 事実と意見と希望を分ける：一次情報/根拠→解釈（論点）→条件付き結論（判断材料）の順で毎回同じ型

            - 視聴者と一緒に"観測"する：「次はここを見る」を提示し、次回で答え合わせ（連載感）
            - 運営者の「予測」を毎回必ず組み込む（当たり/外れを報酬化）
                - 形式：予測（条件付き）→期限→観測ポイント→次回で答え合わせ
                - 当たった時：塾長が素直に喜ぶ＋シンが「珍しく正解」と淡々と褒める
                - 外れた時：塾長が開き直ってギャグ化＋シンが真顔で減点（デッドパン）
                - 注意：外れは"隠さない"。外れを見せて訂正できる＝信頼として積み上げる

        - 狙い：減税を"スローガン"で終わらせず、進捗が追える長編ストーリーにする（次回視聴の理由を作る）

        - 師弟の役割分担
            - 塾長：希望（減税）を"執着ギャグ"として言い続け、熱を絶やさない（連載の旗）
            - シン：毎回「条件は満たした？」「いつまでに？」で折檻し、現実チェック（信頼）を担保
                - 期限：いつまでに何が起きる想定？（会期/選挙/予算/成立時期など）
                - プロセス：実現までの手順はどこまで進んだ？（提出→審議→採決→成立 など"今どこ？"）
                - 数の根拠：可決に必要な人数/合意は足りてる？（与野党/連立/党内の賛否）
                - 一貫性：過去の発言・投票行動と矛盾してない？（ブレ/手のひら返しの検知）
                - 制度の制約：法制度・憲法・省庁手続き・自治体など、構造的な制約に引っかからない？
                - 一次情報：根拠は何？（公式資料/会見/法案/議事録など"ソース提示"を促す）
        - 達成/未達を"イベント化"：達成が近づくほど「Xデー」「最終局面」など章立てを作り、視聴者の期待を積み上げる

- 11. 運用リスク（前提）
    - 出典：運用ガイドライン.md
    - 政治系は誤情報判定・削除圧・分断・支持者心理が複雑に絡む
    - 起きやすいリスク（構造）
        - 政治ニュース自体が"フェイク/誤情報"と疑われやすい
            - 断片・切り抜き・伝聞・時系列ズレがあるだけで「ミスリード」に見えやすい
        - 不都合情報は"権威性/ブランド"の毀損として扱われやすい
            - 発信内容が政治家/政党のブランド価値に影響し得るため、反発・通報・拡散炎上のリスクが上がる
            - 場合によっては、多方面からの働きかけで削除方向に動く可能性もある
        - 支持者の感情（認知的不協和）で反発が起きやすい
            - 事実でも信じたくない（自己同一性の防衛）
            - 事実ではないと否定したくなる（不都合な情報の拒否）
    - 守るべき最重要3点（毎回チェック）
        - 1：誤情報を出さない：出典/日付/時系列/仮説線引き（誤認・削除圧の回避）
        - 2：党派化しない：人物叩きでなく構造主語、条件付き是々非々（分断と通報の回避）
        - 3：学び回収で終わる：整理→判断材料→観測ポイント（疲れさせず習慣化）

- 13. 競合分析の勝ち筋
    - 出典：運用ガイドライン.md、政治系チャンネル分析_伸びる企画と伸びない企画の構造.md
    - 競合の強みは伸ばす

    - 競合の弱みは潰す
        - 事実誤り・制度説明ミスを避ける
        - 代替案・判断材料（条件）を渡す
        - 終盤で普遍的な学びに回収する
    - なぜ政治×エンタメの相性が良いか
        - 政治は現在進行形だからこそ、ギャグ要素との相性が良い
        - 深刻なトーン一辺倒だと視聴者は"解決不能な現実"に直面してしまう

        - リッチなサムネで「番組感」「網羅感」「没入感」を訴求する
        - 派手な編集と台本のギャグ性で政治の重さを緩和する
        - 伸びない企画は全て「実益の欠如」と「不安要素（安心感の欠如）」に集約される

                - 競合：一人語り。視聴者に"授業されてる感"が出やすい
                - 自ch：塾長（エンタメ/脱力/ボケ）×シン（信頼/根拠/ツッコミ）の分業。「盗み聞き感」で敷居が下がる
            - 4. 運営者の"立場"が明確
                - 競合：全方位いじり＝「何を信じているか」が見えにくい
                - 自ch：「減税（実益）」という一貫した軸を持ち、どの党でも"実現条件"で評価
            - 5. 予測→答え合わせの連載設計
                - 競合：基本は単発完結。1本見たら満足で終わりやすい
                - 自ch：毎回「条件付き予測→期限→観測ポイント→次回で答え合わせ」を組み込む
            - 6. 減税の長編ストーリー化
                - 競合：テーマは毎回変わり、チャンネル全体を貫く"縦軸"がない
                - 自ch：減税ミッション/減税ゲージで「見届けたい」という連載視聴の理由を作る
            - 7. コメント欄の設計思想
                - 競合：コメント欄は感想や議論の場（荒れるリスクあり）
                - 自ch：「A/Bどっち？」の小テスト化。次回冒頭で回収
            - 8. IPのギャップ設計
                - 競合：「面白い編集×政治」でエンタメ感を出す（編集依存）
                - 自ch：キャラ自体のギャップ（制服の格×コタツで脱力）。編集に頼らず「余裕のある強者」を表現
        - 補足（既存分析より）
            - 第一ベンチマークの強み：視聴報酬が二重 / 全方位いじりで中立 / 作品性
            - 第一ベンチマークの弱み：事実誤り/制度説明のミスが時々出る / 判断材料が薄い時がある
            - まとめ集/反応集の限界：視聴報酬が"短期の怒り/嘲笑"に寄る。ファンコメントが生まれにくい
            - 参照：競合チャンネル一覧.md / 第一ベンチマーク分析シート.md
    - Company（自社の強み｜リソース・競合との差別化ポイント）
        - クオリティの高い動画を納品できる編集者が揃っており、外注化も得意

- 15. 視聴者の「面白い」の定義（8つ）
    - 出典：政治視聴者.md / 企画・サムネ設計_〇〇感メモ.md
    - 前提：制作者の「面白い（＝娯楽/笑い）」と視聴者の「面白い（＝快感/安心/納得）」がズレると離脱する
    - 8定義
        - 1. 新たな知識を得られたとき：霧が晴れる快感 / 新事実・意外性 / 語れるネタ化 / 点が線になる快感
        - 2. 悩みや不安が解消する：安心感（振り回されない） / 予測できた感 / 出口がある / 不安の言語化
        - 3. 網羅感：これだけ見ればいい完結感 / 整理・要点化 / 学んでる自分の維持
        - 4. 価値観が変わる（パラダイムシフト）：前提が覆る衝撃 / リフレーミング / 多面的理解 / 判断軸の獲得
        - 5. 構造が見えるようになる：腹落ち感 / ストーリーで理解 / 世界の解像度UP / 構造の矛盾を知的に笑える
        - 6. 孤独な価値観に共感される様子：代弁してくれた / 立場の肯定 / 同志がいる安心 / 言語化の代行
        - 7. ワクワクする（知識武装への高揚）：知的優位性 / 成長実感 / 社会的報酬への期待 / 進捗感・参加感
        - 8. 不協和の解消：ツッコミの快感 / ガス抜き / 答え合わせの安心 / 正義感
    - 制作者とのズレが起きやすい3点
        - 1. 制作者が「面白い＝笑い」→視聴者は「面白い＝安心（振り回されない）」
        - 2. 制作者が「知識が増える＝情報量」→視聴者は「語れる形で持ち帰れる」
        - 3. 制作者が「ワクワク＝次回予告」→視聴者は「知識武装している未来への高揚」
- 16. 伸びるワード集（サムネ・タイトル用）
    - 出典：企画・サムネ設計_〇〇感メモ.md / 第一ベンチマーク分析シート.md
    - CTRが上がりやすいワードを〇〇感ごとに分類
    - 1）網羅系（安心感 / 網羅感 / これを見ればいい感）
        - 完全解説 / 完全網羅 / 全史 / 〜の歴史 / 〜史
        - 総まとめ / 徹底比較 / 前編/後編
        - 狙い：「これ一本で全部わかる」＝視聴者は安心感を買っている
        - 実例：「民主党政権全史」「高市早苗史」
    - 2）問い系（知的好奇心 / 腹落ち期待 / 構造が見える感）
        - なぜ〜なのか / 〜って何だ？ / どこに投票する？
        - 結局こういうこと / どんな人か / 何者？
        - 狙い：「問い」がそのまま視聴報酬の約束になる
        - 実例：「結局、何がしたい？」「中道連合なぜ大敗？」
    - 3）安心系（置いていかれない感 / 優しさ / 時短感）
        - 超わかりやすく / 神解説 / ゼロからわかる
        - 誰でもわかる / 徹底図解 / これだけ見ればOK
        - 3分で理解 / 3点で分かる
        - 狙い：堅い政治への警戒心を解く（脱落させない約束）
    - 4）結論系（スッキリ感 / 知的好奇心 / 衝撃）
        - 〜の末路 / 〜の正体 / 崩壊 / 終焉 / 激変
        - 史上最悪 / 前代未聞
        - 狙い：「オチが明確」＝視聴前にゴールが見えるので安心してクリックできる
        - 実例：「地獄の民主党政権」「史上最悪の政権」
    - 5）実益系（見通し立つ感 / 危機回避感 / 予測できた感）
        - 試算 / シミュレーション / 確定 / ロードマップ（計画表）
        - Xデー / タイムリミット / 備え / 処方箋 / 逃げ道
        - 手取りはいくら減る？（生活への翻訳）
        - 狙い：漠然とした不安を「具体的なシナリオ」に変え、心の準備（安心）をさせる
    - 6）ツッコミ系（知的エンタメ感 / 優越感 / スカッと感）
        - 矛盾 / 欠陥 / 詰み / 無理ゲー / バグ / ブーメラン
        - 算数ができない / 物理的に不可能 / 崩壊の数学的根拠
        - 狙い：人格攻撃ではなく「構造の矛盾」を指摘する知的な笑い
        - 注意：単なる悪口に見えないよう「仕組みの欠陥＝学び」として回収
    - 7）衝撃系（スケール感 / 覗き見感 / 重要感）
        - 壮絶すぎる人生 / すべてのはじまり（起源） / 〜年の因縁
        - 史上最大 / 地獄 / 絶滅
        - 実態 / たった1人で○○万人
        - 狙い：「見ないと損しそう」の重要感を作るが、必ず学び・腹落ちで回収
    - NG寄りワード（使うなら回収の約束をセット）
        - 闇 / 炎上 / 裏側 / 〜が起こるって本当？
        - 理由：視聴者は食傷気味。単なる煽り・ゴシップに見えると離脱する
        - 回避策：「構造的な闇（制度の欠陥）」を「学び」として解説するならOK
    - 運用ルール
        - 1枚のサムネで狙うワードは1〜2個（欲張るとメッセージがボケる）
        - ワードが生む〇〇感を意識する（例：「完全攻略」→網羅感/安心感）
        - タイトルとサムネで〇〇感を揃える（認識ズレ防止）
        - 「不安」より「解決（判断軸）」を売る：答えをサムネで約束
        - 「遠い危機」より「近い実益」に翻訳する：×「憲法改正の是非」→○「あなたの徴兵リスクと税金はどうなる？」
