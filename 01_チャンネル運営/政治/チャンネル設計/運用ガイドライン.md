# 政治チャンネル｜運用ガイドライン

目的：誤情報・名誉毀損・著作権・炎上・規約違反・信用毀損を防ぎ、**長期で信頼を積み上げる**ための運用チェックリスト。

関連：
- `IP・キャラ設定.md`（批判の作法・ポジション取り・表現変換）
- `企画・サムネ設計_〇〇感メモ.md`（サムネ/企画の〇〇感設計）
- `競合チャンネル一覧.md`（競合の強み/弱み転用）
- `台本テンプレ_代弁→整理→学び.md`（台本構成の型）

## 0. 前提：政治ジャンルが“デリケート”な理由（なぜルールが必要か）
政治系は、単に著作権だけではなく **誤情報判定・削除圧・分断・支持者心理**が複雑に絡みます。  
そのため「ちょっとした言い回しの違い」でも、炎上・通報・削除・信用毀損につながり得ます。

### 起きやすいリスク（構造）
- **政治ニュース自体が“フェイク/誤情報”と疑われやすい**
  - 断片・切り抜き・伝聞・時系列ズレがあるだけで「ミスリード」に見えやすい
- **不都合情報は“権威性/ブランド”の毀損として扱われやすい**
  - 発信内容が政治家/政党のブランド価値に影響し得るため、反発・通報・拡散炎上のリスクが上がる
  - 場合によっては、多方面からの働きかけで削除方向に動く可能性もある
- **支持者の感情（認知的不協和）で反発が起きやすい**
  - (a) 事実でも信じたくない（自己同一性の防衛）
  - (b) 事実ではないと否定したくなる（不都合な情報の拒否）

### 運用示唆（このガイドラインの使い方）
- **過去→現在の誤認**が最も危険：日付・段階・前提条件を必ず明示する
- **人物/政党の落とし**に見せない：主語を政策/制度/プロセス（構造）に固定する
- **断定を抑える**：事実/仮説/意見を分け、反証条件（外れる条件）も添える
- **削除/通報リスクの分散**：煽り・断罪より「整理→判断材料→観測ポイント」で回収する

## 0-2. 今回のチャンネル設計まとめ（このプロジェクトの設計図｜深掘り版）
※このガイドラインは「事故回避」だけでなく、**チャンネルの勝ち筋（長期で信頼を積む設計）**を実装するための運用書でもある。  
詳細は関連ファイルへ。ここは“1枚で全体像が分かる”ように統合している。

### 1) コンセプト（視聴者に約束する価値）
- 政治を「堅苦しい解説」にせず、「ソファでくつろぎながら賢くなれる感」「映画や特番のようなエンタメ感」を出す。**脱力（ギャグ/ツッコミ）で気軽に見られる**のに、最後は **論点整理で腹落ち・安心**が残る。


### 2) 想定視聴者（心理に刺さる報酬）
- コア：50代前後〜（男性比率高め、30〜40代/女性も一定数）。共通項：メディア不信／深掘り学習欲求／危機感／知的優位性。
- 視聴の快感（＝面白い）：安心感／腹落ち／整理／置いていかれない／語れる／代弁スカッと／見通し／危機回避。  
  → だから「断罪・炎上だけ」は短期で伸びても、長期で疲れて離脱する。
- 参照：`政治視聴者.md`、`企画・サムネ設計_〇〇感メモ.md`

### 3) 仮想敵（何に勝つチャンネルか｜構造寄りにする）
- 増税や社会保障料を引き上げやすい“制度・プロセス”／不安煽りで再生を取る発信／事実不明の断定／難解語で煙に巻く解説／制度だけ叩いて解決策を出さない発信。
- 注意：人物/政党“叩き”が主語になると、分断と通報リスクが跳ね上がる（このガイドラインの全体方針と矛盾する）。
- 参照：`IP・キャラ設定.md`（今回のch設計欄）

### 4) 競合に勝つ方針（短期の面白さ×長期の信頼）
- 競合の強みは伸ばす：テンポ/ギャグで気分が上がる＋政治がスッキリ分かる（視聴報酬の二重化）。
- 競合の弱みは潰す：事実誤り・制度説明ミスを避ける／代替案・判断材料（条件）を渡す／終盤で普遍的な学びに回収する。
- 参照：`第一ベンチマーク_Seiji_Muteki_Oji_分析シート.md`（要約・勝ち筋）

### 5) 企画の柱（伸びやすい型）
- **迷い解決（比較・網羅）**：「どこに投票？」「総まとめ」「徹底比較」など、判断軸が手に入る企画。
- **全史/人物回（時系列で腹落ち）**：「〜の歴史」「人物完全解説」＝網羅感＋信頼＋語れる。
- **未来の備え（予言ではなく試算/シミュレーション）**：AならB（条件）で“備え”を作る。
- **ツッコミ（人格攻撃ではなく構造矛盾）**：計算が合わない／手続きがバグってる等、ロジカルな笑いに変換。
- 参照：`企画・サムネ設計_〇〇感メモ.md`

### 6) サムネ/タイトル設計（〇〇感で設計する）
- サムネは「出来事」ではなく **得られる〇〇感** を約束する装置（安心・ワクワク・人生変わる感）。
- 1枚で狙う〇〇感は1〜2個。タイトルとサムネの〇〇感を揃える（ズレ防止）。
- “リッチさ”は政治の「堅い/面倒」をTV特番化してクリックさせるための演出（重要感・信頼の誤認・没入）。
- 参照：`企画・サムネ設計_〇〇感メモ.md`

### 7) 台本の型（炎上回避しつつ伸ばす）
- 基本形：**代弁 → 整理（なぜ？を構造化） → 学び（判断軸/観測ポイント）**。
- 前提固定：主語は制度/プロセス／時系列明示／出典を残す。
- 参照：`台本テンプレ_代弁→整理→学び.md`

### 8) コメント欄の設計（荒れにくく習慣化する）
- 「議論の場」ではなく「答案提出の場」へ：**A/Bどっち？理由は条件で一行**（小テスト化）。
- 次回冒頭で回収して“参加価値”を作る（習慣化）。
- 参照：`IP・キャラ設定.md`（コメント欄活性・A/B設計）

### 9) IP設計（脱力塾長×シン＝長期で追う装置）
- **脱力塾長**：脱力（コタツ/ソファ/寝転び）×鋭い整理（論点3点）のギャップで“余裕の強者”を作る。  
  口調は無理ゲー/バグ/詰み。自称/生活感でツッコミ誘発。
- **シン（弟子）**：冷静ポーカーフェイス毒舌。代弁・翻訳・ツッコミ・折檻で、塾長の脱線を現実に戻す（信頼担保）。
- **減税の連載化**：減税ミッション/減税ゲージで進捗を見せ、毎回観測ポイントで追えるようにする。
- **予測→答え合わせ**：当たりは喜び、外れはギャグ＋訂正で信頼を積む（視聴継続の二重報酬）。
- 音声（現状）：塾長＝ElevenLabs（ランビール×イレブンv3）／シン＝ElevenLabs（樹×イレブンv3）。  
  詳細は `IP・キャラ設定.md` に固定。
- 参照：`IP・キャラ設定.md`

### 10) このガイドラインで守るべき“最重要3点”（毎回チェック）
1. **誤情報を出さない**：出典/日付/時系列/仮説線引き（誤認・削除圧の回避）
2. **党派化しない**：人物叩きでなく構造主語、条件付き是々非々（分断と通報の回避）
3. **学び回収で終わる**：整理→判断材料→観測ポイント（疲れさせず習慣化）

## 0-3. 3C分析

### Customer（市場・視聴者｜視聴者のニーズ・悩み）
- 政治解説は偏向報道や叩き系の動画が多く、政治家のネガティブな面ばかりのチャンネルで不安を煽られているため疲れている。
- 政治の知識をつけて、将来の不安を解消するような、知識をつけたい。
- 安心感を得たい。
- 政治という堅苦しい内容を面白く、わかりやすく解説された動画を見たい
- だけど、TVでは政治献金やスポンサーなどもあるため、偏向報道しかされないためTVは信用できない
- **補足**：コア視聴者は50代前後〜（男性比率高め）。怒り発散より不安の制御。「学んでる自分」でいたい。〇〇感のコア＝安心感/腹落ち感/整理感/置いていかれない感。
- 参照：`政治視聴者.md` / `企画・サムネ設計_〇〇感メモ.md`

### Competitor（競合｜競合の量と質・共通点と弱点・差別化ポイント）
- 政治系のYouTube運営者は情弱層と、レベルが高い方の差が激しい印象。
  - →なぜなら、政治で一般的なチャンネル設計法として、まとめ集/反応集や、切り抜き系で稼いでいる人が多いから。
- また、ショートや8分尺などの長尺の中の短尺動画を上げる層が多く、30〜40分の長尺台本を作り込める人がライバルにいない。
- 第一ベンチマークでは、ギャグの精度は高い。しかし、視聴者が本当に求めている本質「安心感」「網羅感」「今後の解決策を知りたい」が捕捉されきっていない。
- IPが見せきれていない
- 視聴者意見の代弁をするようなキャラがいない→掛け合い形式にしてシンを視聴者意見の代弁とする
- 視聴者側が見られている感、授業されている感が強い→理由は、一人語りのため運営者が直接視聴者に語りかけている感が強い。現代人は、直視されることを避けている（ウーバーイーツの普及（店に入りたくない、頼んでいる様子を見られたくない、一蘭の制度が流行っている理由））ため、居酒屋の会話を盗み聞きしている感で動画を見続けたい。
- 無敵おじさんは中立視点に寄りすぎて、運営者の立場がわかりにくい→ここは脱力塾長の長期目標の設定や、仮想敵を設定して対処
- IPの魅力度、IPの魅力の本質はギャップです。当チャンネルでは、IPのギャップや動画編集と台本のギャップを作りたいです。

## 1. 出典ルール（一次情報の扱い）

政治系の視聴者は「一次情報」を求めやすい。設計段階からルール化する。

### チェックリスト
- [ ] チャンネル/動画/記事のURLを必ず記録する（競合・参考・一次情報）
- [ ] "いつの話か"を残す（年・月、できれば日付）
- [ ] 断定できない場合は「仮説」として線引きする
- [ ] 公式資料/会見/法案/議事録など"ソース"を明示できる状態にする
- [ ] 台本/概要欄に出典を記載する（視聴者が自分で確認できるようにする）

### NG
- 出典不明のまま断定する
- 伝聞・SNS投稿のみを根拠にする（裏取りなしの情報拡散）
- 他チャンネル/メディアの主張を自分の分析として流用する

## 2. 時系列の正確性

政治は時系列のズレが誤情報に直結する。「いつ」を常に明示する。

### チェックリスト
- [ ] 「いつの話か（年月日）」を台本・テロップに明示する
- [ ] 過去の出来事を現在進行形で語らない（誤認を防ぐ）
- [ ] 法案・政策の「提出→審議→採決→成立→施行」のどの段階かを明示する
- [ ] 時系列に沿って因果を整理する（結果→原因の順で語る場合も、時系列を補足する）
- [ ] 情報の鮮度を明示する（例：「2026年2月時点の情報です」）

### NG
- 古い情報を"最新"として扱う
- 時系列を曖昧にしたまま因果関係を主張する
- 法案の段階（審議中/成立済み）を省略して断定する

## 4. 政治色のコントロール（炎上・分断の回避）

政治系は信用を落とすと回復が極めて難しい。「中立」ではなく「条件付き是々非々」を軸にする。

### 基本スタンス
- **特定政党の信者ではなく「政策（条件）」の支持者**として振る舞う
- 人物叩き・人格攻撃ではなく、**政策・制度・意思決定プロセス（構造）**を主語にする
- 事実と意見と希望を分ける（一次情報/根拠 → 解釈/論点 → 条件付き結論/判断材料）

### チェックリスト
- [ ] 批判は「人格」ではなく「構造・制度・結果」に向いているか
- [ ] 特定政党/政治家への過度な応援・批判に見えないか
- [ ] 「支持者が〜」など相手陣営を丸ごと煽る表現がないか
- [ ] 対立煽りより「論点の棚卸し」に寄せているか
- [ ] 視聴者に善悪ジャッジ（裁判官役）を強要していないか
- [ ] 運営者の意見は「条件付き（AならB）」で提示しているか

### 表現の変換テクニック（ギャグ化）
- **「怒り」→「呆れ/趣味」**
  - 例：「また始まりました『増税チャレンジ』もはや日本の趣味です（笑/呆れ）」
- **「不信感」→「伝統芸能/お約束」**
  - NG：「発言がコロコロ変わって信用できない」
  - OK：「出ました、伝統芸能『手のひら返し』！このスピード感だけは超一流です（笑）」

### NG
- 「〇〇議員はバカだ」「〇〇の政党はダメだ」（人格攻撃）
- 「〇〇党は終わってる/最高」（党名での評価断定）
- 「支持者が〜」（陣営丸ごと煽り → コメント欄の場外乱闘化）
- 善悪ジャッジを強要する企画（視聴者が疲れる）

## 5. 学び回収（信頼を積む終盤設計）

不安/怒りを煽るだけで終わらせず、最後に「整理→判断材料→観測ポイント」まで渡す。

### チェックリスト
- [ ] 動画の終盤で「結局こういうこと（要約）」を提示しているか
- [ ] 視聴者が「次に何を見ればいいか（観測ポイント/チェックリスト）」を持ち帰れるか
- [ ] 感情報酬（スカッと/怒り）だけでなく、理解報酬（腹落ち/安心）を回収しているか
- [ ] 特定政党の応援/批判ではなく「条件」「判断基準」として出しているか

### 学び回収の型（台本テンプレ連動）
- 学び（結論）：＿＿＿＿
- 次に見るポイント：① ＿＿ ② ＿＿ ③ ＿＿
- 締め：「今日の学びは＿＿。次は＿＿を見ればOKです」

### NG（学び回収なしの危険）
- 闇/裏側/炎上だけで終わると「不安増幅→疲労→離脱」になる
- 煽りだけの動画は短期では伸びても、長期で信用が積まれない
- 感情報酬に偏ると「見た後に疲れる→習慣化しない」

## 6. 意見テンプレ（事実/意見/希望の分離）

運営者の意見を出す際の安全フォーマット。

### 構成
1. **事実（一次情報）の提示**：今回の争点はこれ。決まっている事実はここまで。
2. **論点の棚卸し（判断軸を3つ）**：「家計」「将来」「制度（持続性）」など"党から独立した軸"にする
3. **運営者の意見（条件付き）**：「私は"党"じゃなくて"条件"で選ぶ。条件Aが満たされるならA寄り、満たされないならB寄り」
4. **投票（コメント誘導）**：「あなたの結論はA/Bどっち？理由は一行で（条件で）」
5. **回収**：コメントの多かった条件を次回の冒頭で読み上げて"参加価値"を作る

### 使える宣言（党派に見せないための言い回し）
- 「私は〇〇党のファンじゃなくて、『条件（政策/数字/手続き）』のファンです」
- 「推し政党の話じゃなくて、"この条件が満たされるなら賛成"の話をします」
- 「今日は"誰が正しいか"より、"どの判断軸で選ぶか"だけ持ち帰ってください」

## 7. 誤情報リスク

### チェックリスト
- [ ] 主張に裏付け（出典/データ/一次情報）があるか
- [ ] 仮説と事実を明確に区別しているか
- [ ] AIが生成した情報を鵜呑みにしていないか（一次情報は人間が確認）
- [ ] 誤りに気づいたら速やかに訂正/補足を出す（隠さない＝信頼の積み上げ）

### 訂正の運用
- 間違いを隠さず訂正し、学習して前進する姿を見せる（長期の信頼資産になる）
- 訂正動画/コミュニティ投稿で対応する
- 外れた予測はギャグ化＋訂正で回収する（信頼の二重取り）

## 8. 名誉毀損リスク

### チェックリスト

## 10. 炎上・信用リスク（長期の敗北条件チェック）

長期で信用を失う要因を設計段階で潰す。

### 敗北条件チェック（企画/台本ごとに確認）
- [ ] **サムネ/タイトルの約束を回収しているか**（約束の裏切り＝信用毀損）
- [ ] **根拠が薄くないか**（「それっぽい話」に見えると信用が積めない）
- [ ] **前提条件が曖昧でないか**（対象/期間/定義/比較条件が曖昧→誤解→炎上/不信）
- [ ] **刺激過多になっていないか**（煽り/断定/攻撃/不安増幅が強い→疲労→習慣化しない）
- [ ] **コメント欄が攻撃的/分断的にならない設計か**（長期の離脱要因）
- [ ] **制作コストが持続可能か**（運用破綻＝継続不能）

### 勝ち筋チェック（信用を積むための回収）
- [ ] 見終わった後に「結局こういうこと」が残るか（行動/判断/理解のどれか）
- [ ] 出典/データ/引用など裏付けを残しているか
- [ ] 前提の明確化（対象・条件・期間・定義・比較軸）を先に置いているか
- [ ] 反証/例外/反対意見を扱い、断定の押し付けにしていないか
- [ ] 毎回同じフォーマット（結論→理由→根拠→まとめ→次の観測/宿題）で安心して見られるか
